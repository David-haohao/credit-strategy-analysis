# 阶段 0：数据、产品与分析确认（五段主流程）

## 目的

在任何规则挖掘前确认数据能回答哪个产品、哪个场景、哪个分析单位上的策略问题。未确认即停止，不以字段名或行业经验推断。

## 必须确认

| 类别 | 内容 |
|---|---|
| 产品与场景 | 产品类型、还款方式、首借/复借/准入等分析场景 |
| 数据与样本 | 输入路径、样本口径、时间范围、是否有旧策略结果 |
| 分析单位 | 分析颗粒度、`id_cols`、跨颗粒度聚合规则 |
| 表现 | 标签、正样本、坏样本定义、成熟规则和日期字段 |
| 字段映射 | ID、标签、日期、特征、金额、旧策略通过/拒绝字段 |
| 金额 | 是否计算金额指标；如计算，确认分子、分母和过滤 |

读取 input 后可提出字段映射候选，但必须由用户确认。`sample_id` 不能替代确认的 `id_cols`。

## 数据检查

1. 校验确认字段存在、分析单位非空且唯一。
2. 校验标签可表达为二分类，成熟和未成熟样本可区分。
3. 校验候选特征不含 ID、标签、表现结果、金额结果、审批结果和拒绝码等保护列。
4. 记录样本量、时间覆盖、缺失率、标签分布和旧策略字段可用性。

## 输出

- `00_data_check.csv`：字段、类型、缺失、唯一性和映射状态。
- `00_analysis_config.yaml`：用户确认的产品、数据和分析配置。
- `00_manifest.json`：确认凭证、输入路径、分析单位、样本量和阻塞项状态。

只有所有阻塞项已确认，才能进入阶段 1。
