# 阶段 0：数据、产品与分析确认

## 适用与阻断条件

用于任何规则策略分析的开始。它解决“分析什么、按什么单位计算、哪些字段能用”的事实确认，不负责挖掘规则。产品、场景、样本、颗粒度、字段映射、标签、成熟规则、金额口径或用户确认凭证任一缺失时阻断后续阶段。

不得把行业常见做法当作本次事实。例如循环贷可能以借据或支用生成 0/1 标签，而贷前特征通常在人层；是否聚合、如何聚合必须由用户确认。

## 最小输入

| 输入 | 必须确认的内容 |
|---|---|
| 产品信息 | 产品形态、循环/非循环、还款方式、额度与借据/支用关系 |
| 分析场景 | 首贷准入、复贷准入、调额、存量策略复盘或其他明确场景 |
| 原始数据 | 数据路径、编码、字段名、字段类型、时间范围和可用样本量 |
| 样本与单位 | 样本总体、分析颗粒度、`id_cols`、标签/特征/输出颗粒度 |
| 标签与表现 | 目标字段、正样本、坏样本定义、成熟过滤、排除值 |
| 字段映射 | 语义字段到原始列的用户确认映射及未映射字段清单 |
| 金额与旧策略 | 金额分子/分母/过滤，旧策略决策字段及其可用范围（如有） |

## 方法步骤

1. 记录产品形态和业务阶段，说明申请、额度、借据、支用、还款之间的关系；不要从字段名推断产品模式。
2. 确认分析问题是“发现候选拒绝规则”“组合规则”还是“评估既有新策略”；同一运行只选择一个任务类型。
3. 读取数据的列名、类型、空值、时间范围与 ID 重复情况，仅生成候选映射供用户确认，不直接执行候选映射。
4. 与用户确认样本总体和成熟规则。未成熟、未知或被排除标签不得参与表现指标分母。
5. 确认分析颗粒度与 `id_cols`，据此生成 `analysis_unit_id` 并验证唯一性。不能以 `sample_id` 替代。
6. 标签、特征或输出颗粒度不同，停止并要求用户给出聚合规则，例如观察窗、首笔、最近一笔或任一坏账；记录规则的业务含义。
7. 确认字段映射和候选特征。把 ID、标签、表现字段、金额结果、审批/拒绝结果、日期及未经批准的衍生字段放入保护列。
8. 如需金额指标，确认分子、分母、公式、成熟/通过过滤和汇总颗粒度；金额口径不完整时只输出计数类指标。
9. 写入确认凭证、运行配置、数据检查结果和 `run_manifest.json`，作为后续阶段唯一事实来源。

## 数据检查判定

| 检查 | 通过条件 | 不通过时的处理 |
|---|---|---|
| 字段映射 | 必需语义字段均有用户确认的原始列 | 列出候选映射并停止，不能自动选列 |
| ID 完整性 | `id_cols` 非空、无空值、组合后唯一 | 返回重复样本和 ID 缺失明细 |
| 标签可用性 | 标签字段、正样本、坏样本和成熟规则均已确认 | 不计算任何表现或 Lift 指标 |
| 时间一致性 | 样本时间与成熟观察期可解释 | 标出未成熟或时间穿越样本 |
| 特征可用性 | 字段在策略决策时点可获得 | 移至保护列并记录泄漏风险 |
| 金额一致性 | 分子、分母、过滤与汇总颗粒度完整 | 仅输出计数类结果 |

字段映射应至少保留“业务语义、原始列、数据类型、可用时点、确认人/确认时间、备注”。候选映射只用于提问和审计，不能写入可执行特征清单。

## 候选参数

以下项目必须由用户选择并写入确认凭证；表中示例不是默认值。

| 参数 | 可确认的候选 | 影响 |
|---|---|---|
| 产品形态 | 循环贷、非循环分期、额度产品或自定义 | 决定标签和聚合讨论范围 |
| 分析颗粒度 | 客户、申请、借据、支用、客户-时间窗 | 决定唯一性、样本数和指标分母 |
| 跨颗粒度聚合 | 任一坏账、首笔、最近一笔、观察窗或自定义 | 决定标签和特征如何对齐 |
| 样本总体 | 申请、通过、放款、成熟放款、拒绝或混合 | 决定可观测表现范围 |
| 成熟规则 | 明确的 MOB、账龄、观察窗或标志字段条件 | 决定表现样本 |
| 金额口径 | 确认的分子、分母、过滤和公式 | 决定金额风险指标 |

## 输出表

| 文件 | 最少字段 |
|---|---|
| `data_profile.csv` | 原始列、类型、非空率、唯一值数、候选语义、映射状态 |
| `analysis_unit_check.csv` | 颗粒度、`id_cols`、记录数、唯一单位数、重复数、处理状态 |
| `confirmation_receipt.json` | 用户确认的产品、场景、映射、颗粒度、标签、成熟、金额和聚合 |
| `input_config.yaml` | 本次任务类型、输入路径、确认凭证路径和参数 |
| `run_manifest.json` | 输入、确认凭证、数据校验、输出文件与时间戳 |

## 稳定性与可观测性检查

- 比较数据时间范围与成熟观察窗，确认不会将未成熟样本混入表现分析。
- 发现 `id_cols` 为空、ID 为 null、单位不唯一或跨颗粒度无聚合规则时阻断。
- 字段候选映射、金额字段歧义、旧策略字段覆盖不全时仅报告问题，不代替用户选择。
- 若旧策略拒绝样本没有后续表现，提前标记其在 `swap_in` 分析中不可直接观测。

## 验收不变量

- 每一项业务和数据口径可回溯至用户确认凭证。
- `analysis_unit_id` 由确认的 `id_cols` 构造并在当前颗粒度唯一。
- 任何跨颗粒度转换都有明确聚合规则。
- 保护列不会进入候选特征。
- 未确认金额口径不产生金额风险结论。
