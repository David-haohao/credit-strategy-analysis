# 表现漂移监控

## 最小输入

输入为确认的标签、成熟规则、时间字段、分析颗粒度和金额口径。监控通过率、坏账率、金额风险或路由表现时，基准/对比窗口、样本范围和标签观察期必须一致或明确可比性限制。

## 计算顺序

1. 按确认窗口和分析单位生成实时总体与成熟表现总体。
2. 分别计算通过率、拒绝率、成熟坏账率、金额风险和确认的分层指标。
3. 比较窗口间水平、样本量、成熟覆盖率和分母变化；必要时按路由或产品拆分。
4. 将实时变化、成熟表现变化和不可观测部分分开说明，避免把表现期差异解释为策略变化。

## 输出与披露

输出 `performance_drift_summary.csv` 和分层明细，包含窗口、样本量、成熟量、通过/拒绝率、坏账率、金额指标、可观测性和变化值。披露标签定义、成熟规则、金额公式和分母。

## 验收

表现比较只使用确认的成熟样本；金额指标分子、分母和过滤一致。窗口样本不足、成熟覆盖不一致或标签定义变化时必须标记不可比，而非输出结论。
