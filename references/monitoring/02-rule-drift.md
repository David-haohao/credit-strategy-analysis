# 规则漂移监控

## 最小输入

输入为确认的规则版本、规则顺序、时间字段、分析颗粒度、基准与对比窗口。规则版本、字段语义或聚合规则变化时必须先分版本，不得将变化后的结果称为同一规则漂移。

## 计算顺序

1. 以同一版本规则在基准与对比窗口生成独立命中和顺序漏斗结果。
2. 比较规则命中率、漏斗层级占比、通过率、拒绝码分布和覆盖样本量。
3. 对成熟样本另计算滞后表现趋势，并与实时命中指标分开呈现。
4. 对显著变化按确认路由、产品或字段缺失情况拆分排查，不自动断言原因。

## 输出与披露

输出 `rule_drift_summary.csv` 和层级/规则明细，包含规则版本、窗口、样本量、命中率、变化、通过率和成熟表现状态。报告中区分实时可观测指标与滞后风险指标。

## 验收

比较两侧必须使用同一规则版本、顺序、分析颗粒度和过滤条件。漏斗仍满足层级不变量；未成熟表现不得混入成熟坏账率趋势。
