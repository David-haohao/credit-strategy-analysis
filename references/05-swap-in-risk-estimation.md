# Swap-in 风险预估

用于在旧策略拒绝、新策略通过的 `swap_in` 客群没有直接贷后表现时，生成可审计的离线风险估计。估计不是直接观测，不得与成熟坏账率混写；线上或灰度实验不属于本技能执行范围。

## 先决条件

必须由用户确认：

- 真实坏样本定义、观察窗和成熟规则；
- 历史成熟通过样本、申请时间、申请主键及与 `swap_in` 的同一分析颗粒度；
- 仅在申请/决策时点可得的特征或三方分；
- 训练时间窗、OOT 窗口、分箱/压力参数和输出分层；
- 对无共同支持或三方分缺失/越界 `swap_in` 的处理方式。

不得把旧决策字段、审批结果字段、拒绝原因、既有风险模型结果、授信建议或未成熟贷后字段作为真实风险标签或预估特征。缺少成熟表现时，只能输出不可观测状态和数据缺口。

## 方法选择

| 方法 | 适用范围 | 输出定位 |
|---|---|---|
| 三方分分箱映射 | 强三方分在成熟通过样本上有稳定区分，且覆盖 `swap_in` | 默认 V1 主估计 |
| 局部外推 | 仅一条规则发生明确、小幅阈值放松 | 规则级敏感性分析，不用于多规则级联总体 |
| Parceling / fuzzy augmentation | 需要对拒绝样本选择偏差给出保守情景或模型 challenger | Base / Medium / Stress 区间 |
| 接受倾向诊断 | 通过样本与 `swap_in` 存在足够共同支持 | 判断可外推性；非默认主估计 |
| 随机化策略实验表现 | 已取得合规的真实成熟表现 | 最终校准真值；本技能只记录其离线输入/解释边界 |

旧策略通过率很低、接受概率接近 0、或 `swap_in` 落在历史通过样本未覆盖区时，接受倾向加权不稳定。将这些单位标记为“不可可靠外推”，不得用极端权重强行生成 PD。

## V1：三方分分箱映射

1. 仅在确认的开发期成熟通过样本上拟合三方分箱；固定分箱后应用到 OOT 和 `swap_in`。
2. 每箱输出成熟样本、好/坏样本、观察 bad_rate、置信区间、三方分覆盖和稳定性警告。
3. 映射 `swap_in` 至固定箱，计算：

```text
expected_bad_j = swap_in_count_j * estimated_bad_rate_j
expected_bad   = sum(expected_bad_j)
estimated_br   = expected_bad / sum(swap_in_count_j)
```

4. 对稀疏箱执行用户确认的合并或平滑；不得将零坏样本写为零风险。
5. 将无分数、越界或无成熟通过样本覆盖的客户单列为不可外推，不计入已估计总体。

## 保守性与拒绝推断

将分箱映射作为 Base 估计。若用户确认需要选择偏差敏感性分析，额外输出：

- Medium / Stress：按确认的分数箱压力上调规则进行 Parceling；
- Fuzzy augmentation：拒绝样本按确认的箱级好/坏概率加权进入模型 challenger；
- 接受倾向：使用全申请的旧通过标识建模，仅检查共同支持、概率校准和权重稳定性；只有这些检查通过时才允许作为加权修正。

不得根据估计结果重新选择规则、调节阈值或宣称真实坏账表现。

## 局部规则外推

只在旧/新策略差异由单个特征阈值定义时使用。新增客群必须能表示为明确的阈值区间；在该区间邻域的成熟通过样本上比较固定分箱 bad_rate 或用户确认的局部单调/Logistic 拟合。

多规则 OR、顺序级联、交互路径或同时变更多个阈值时，禁止将单变量线性关系作为整体 `swap_in` 风险结论。

## 输出与验收

至少输出：

| 文件 | 最少内容 |
|---|---|
| `swap_in_risk_estimate.csv` | 方法、分层、样本量、预估坏客户数、预估 bad_rate、区间/情景、可外推性 |
| `swap_in_coverage.csv` | 三方分/特征覆盖、无支持样本、缺失和越界情况 |
| `swap_in_estimation_manifest.json` | 成熟口径、训练/OOT 窗口、方法、参数、假设、来源产物和限制 |

- Base、Medium、Stress 与直接观察指标分列展示。
- 每个估计值记录训练总体、分母、覆盖率和假设。
- `swap_in` 实际表现可用后，仅用固定的分箱/模型重新校准；不得把估计值当作真实标签。
