# HTML 报告

## 最小输入

输入为已完成任务的 `run_manifest.json`、已验证的汇总/明细输出和报告范围。报告不得从对话记录、字段名称或行业惯例补齐口径；缺少 manifest、确认凭证或关键输出时停止。

## 计算顺序

1. 读取 manifest 中的任务、样本、颗粒度、字段映射、标签、成熟、金额和确认凭证。
2. 读取已完成模块的结果，不重新计算指标或改变分母。
3. 按任务呈现背景、样本、方法、核心结果、分层结果、限制和备选建议。
4. 对估计、不可观测、未成熟、样本不足、漂移和过拟合风险单独生成披露。

## 输出与披露

输出 UTF-8 HTML，最少包含任务背景、确认口径、样本范围、时间窗口、核心指标及分母、规则/截断/策略明细、直接观测与估计区分、风险限制和离线备选建议。不得包含上线步骤或审批结论。

## 验收

报告中的口径必须可回溯至 manifest，图表和表格的样本量与汇总 CSV 一致。所有估计值、零分母、未成熟或不可观测结果均有明确标签，且不被写成事实结论。
