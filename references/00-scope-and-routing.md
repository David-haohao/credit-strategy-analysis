# 范围与路由卡

本卡是所有任务的唯一必读入口。它只识别任务、列出阻塞项和指定后续最小阅读集；不重复数据契约和指标公式。

## 范围

本技能只做离线规则、截断、策略、监控和报告分析。不做生产上线、灰度、流量分配、线上规则部署或审批材料。任何业务口径均由用户确认，常见做法只能作为候选提示。

## 阻塞项

| 情况 | 必须由用户确认 |
|---|---|
| 所有任务 | 产品形态、还款方式、场景、任务类型、样本口径、分析颗粒度、`id_cols`、字段映射 |
| 表现或风险指标 | 标签、正样本、成熟规则、标签/特征/输出颗粒度及聚合规则 |
| 金额指标 | 分子、分母、样本过滤 |
| 分数截断 | 分数字段、风险方向、缺失分处理、业务约束 |
| 规则回测 | 规则配置或拒绝码、规则顺序、规则缺失值语义 |
| 策略模拟 | 策略配置、通过/拒绝语义、路由覆盖和兜底处理 |
| Swap | 旧策略结果、新策略结果、不可观测人群的估计方法 |
| 监控 | 基准窗口、对比窗口、规则或分箱版本、告警线 |

`sample_id` 不是默认分析单位。循环贷中的借据、支用、客户聚合，以及任何字段候选映射，均不得自行推断。

## 路由与按需读取

按路由只读取任务所需的 reference；不得默认通读 `02-data-contract.md` 或 `03-metric-definitions.md`。

| 任务 | 必读方法文件 | 按需读取的契约或指标 |
|---|---|---|
| 产品或场景不清楚 | `01-product-and-scenario.md` | `02` 的颗粒度与字段映射部分 |
| 规则挖掘 | `rules/01-rule-mining.md` | `02` 特征与标签；`03` 样本、表现、单规则、WOE/IV |
| 规则回测 | `rules/02-rule-backtest.md` | `02` 规则字段；`03` 样本、表现、单规则 |
| 规则漏斗或组合 | `rules/03-rule-funnel.md`、`rules/04-rule-combination.md` | `03` 单规则和漏斗 |
| 分数截断 | `strategy/01-score-cutoff.md` | `02` 分数字段；`03` 表现、金额、截断、KS/AUC |
| 策略模拟 | `strategy/02-strategy-simulation.md` | `02` 策略字段；`03` 样本、表现、金额 |
| Swap 或分路由 | `strategy/03-swap-analysis.md`、`strategy/04-routing-strategy.md` | `03` Swap、表现、金额 |
| PSI 或漂移 | `monitoring/01-psi-monitoring.md`、`02-rule-drift.md`、`03-performance-drift.md` | `03` PSI、监控、表现 |
| HTML 报告 | `reporting/01-html-report.md` | 已完成模块的 manifest 和相关指标章节 |
| 只问指标 | `03-metric-definitions.md` | 只读被询问的章节 |

## 最小执行顺序

1. 识别主任务；多任务时先选择能为后续任务提供输入的主任务。
2. 依据上表提出该任务的阻塞问题；缺失则停止。
3. 读取 input 后可生成字段映射候选，但只能用于提问。
4. 用户确认后写入确认凭证和运行配置。
5. 按路由读取最小方法、契约和指标集合，运行 `validate_contract` 后再进入任务脚本。

## 停止规则

字段映射、分析颗粒度、跨颗粒度聚合、标签或成熟规则、金额口径、分数方向、旧/新策略结果、路由字段或监控窗口任一必需项缺失时，停止并向用户追问。不得以行业惯例、字段名称或产品形态补齐。
