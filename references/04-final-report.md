# 阶段 4：最终报告

## 适用与阻断条件

用于将前四阶段已经生成且可追溯的产物汇总为最终 HTML 报告。它不重新计算 IV、Lift、级联、策略效果或 Swap，也不从字段名、对话记录或行业惯例补猜缺失口径。

缺少阶段 manifest、确认凭证、关键汇总表或可观测性披露时停止。报告可以形成证据支持的结论与风险披露，但不自动产生上线决策。

## 最小输入

- 阶段 0 的确认凭证、输入 manifest、数据检查和样本口径。
- 阶段 1 的特征筛选、分箱 IV、候选规则和稳定性输出。
- 阶段 2 的 Lift 排名、TopN、级联漏斗和 OOT 输出。
- 阶段 3 的策略效果、Swap 汇总、分层明细、金额口径和可观测性声明。
- 确认的报告范围、标题、时间窗口和必要脱敏规则。

## 方法步骤

1. 读取并校验前四阶段 manifest 的任务、确认凭证路径、分析颗粒度、标签、成熟规则、金额口径和输出路径是否一致。
2. 读取已有汇总表和明细表，验证报告中的样本量、分母和规则数可回溯至原始产物；不重新计算任何指标。
3. 按固定叙事组织：分析目标与产品场景、确认口径与数据质量、单规则证据、TopN 级联策略、整体效果、Swap、风险与结论。
4. 将开发/OOT 稳定性、规则重叠、样本不足、未成熟、金额口径限制和 `swap_in` 估计/不可观测项目单独披露。
5. 对所有图表和表格标注样本范围、时间窗口、分母和观测类型；直接观测、估计和不可观测必须使用不同标签。
6. 输出 UTF-8 HTML 和报告 manifest，记录被引用的文件、生成时间和未解决问题。

## 报告章节与结论规则

| 章节 | 必须回答的问题 | 证据来源 |
|---|---|---|
| 分析背景 | 哪个产品、场景、总体和时间窗被分析 | 阶段 0 确认凭证与 manifest |
| 数据与口径 | 分析单位、标签、成熟和金额口径是什么 | 阶段 0 数据检查 |
| 单规则证据 | 哪些特征和规则通过筛选，稳定性如何 | 阶段 1 候选与 OOT 输出 |
| 组合策略 | 为什么选择该 TopN，逐层边际贡献如何 | 阶段 2 排名与漏斗 |
| 策略效果 | 通过/拒绝、风险、捕获和误伤如何变化 | 阶段 3 效果汇总 |
| Swap | 哪些变化可直接观测，哪些为估计或不可观测 | 阶段 3 Swap 输出 |
| 结论与限制 | 支持何种策略取舍，哪些风险仍未解决 | 已有产物与披露 |

结论只能描述已验证的取舍，例如“在确认成熟总体内，新策略提高了坏客户捕获，同时增加了好客户误伤”。不得把估计 `swap_in`、未成熟样本或未确认金额口径写成确定事实；不得使用“应上线”“自动放量”或等价审批措辞。

## 候选参数

| 参数 | 必须确认的内容 | 作用 |
|---|---|---|
| 报告范围 | 产品、场景、时间窗、分析总体 | 限定结论边界 |
| 展示层级 | 总体及确认的产品/时间/客群分层 | 防止选择性呈现 |
| 指标展示 | 样本与金额指标、分母、格式 | 保持口径透明 |
| 脱敏 | 规则字段名、样本标识和金额展示规则 | 控制输出范围 |
| 结论措辞 | 证据结论与风险披露格式 | 不替代上线决策 |

## 输出表

| 文件 | 最少内容 |
|---|---|
| `final_report.html` | 场景、确认口径、数据质量、规则证据、TopN 级联、策略效果、Swap、结论和风险披露 |

## 稳定性与可观测性检查

- 报告中规则数量、TopN、通过/拒绝量、风险和 Swap 样本量必须与阶段产物一致。
- 估计和不可观测项目必须显式标记，不能与直接观测值汇总或使用同一结论措辞。
- 缺少成熟规则、金额过滤、分母、样本范围或来源文件的图表/表格不得生成。
- 若开发和 OOT 表现方向不一致，报告必须同时展示而不能只选取更优窗口。

## 验收不变量

- 报告只消费既有产物，不重新计算。
- 所有指标、口径和结论均可回溯至确认凭证与 manifest。
- 报告包含策略取舍结论和风险披露，但不自动产生上线决策。
- HTML、CSV 和 JSON 均为 UTF-8，且缺失来源时明确阻断。
