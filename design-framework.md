# 信贷规则策略分析设计框架

## 1. 框架目的

本框架用于把已经确认的数据和业务口径，转化为可审计的离线准入规则策略分析。固定流程为：

1. 数据与产品确认。
2. 单规则挖掘。
3. Lift TopN 规则组合。
4. 策略效果与新旧 Swap 评估。
5. 最终报告。

它不是评分卡建模框架，也不负责监控、分数截断、线上路由或上线审批。

## 2. 共同原则

- 所有业务口径以用户确认凭证为准，配置必须引用该凭证。
- 分析单位由确认的颗粒度和 `id_cols` 决定，不能默认使用 `sample_id`。
- 标签、特征和输出颗粒度不一致时，用户必须确认聚合规则；尤其不得自行假设循环贷的借据/支用标签如何汇总到客户。
- 规则只可使用贷前可获得字段。标签、表现期字段、金额结果、审批结果、拒绝码、ID 和日期默认是保护列。
- 所有比例必须显示分子、分母和样本范围；零分母输出空值，不用零替代。
- 所有参数均为用户确认的运行时配置，不设置隐含行业默认值。

## 3. 阶段产物与依赖

| 阶段 | 输入 | 核心产物 | 进入下一阶段的条件 |
|---|---|---|---|
| 0 数据与产品确认 | 原始数据、产品说明、用户确认 | 确认凭证、配置、数据检查、manifest | 所有阻断项已确认且分析单位唯一 |
| 1 单规则挖掘 | 成熟样本、候选特征、阶段 0 产物 | 特征筛选、分箱明细、单规则与 CART 路径候选 | 每条候选满足确认的命中限制与稳定性检查 |
| 2 规则组合 | 合格候选规则、确认 TopN | Lift 排序、级联拒绝漏斗、组合规则 | 组合语义和边际结果可复现 |
| 3 策略评估与 Swap | 新策略结果、旧策略结果（如有） | 整体效果、Swap 矩阵、观测边界 | 所有指标和估计假设已披露 |
| 4 最终报告 | 前四段产物和 manifest | UTF-8 HTML 报告 | 仅引用可追溯产物，不重新计算 |

## 4. 单规则的判定逻辑

先以 IV 做特征筛选，再对通过筛选的字段生成可解释候选：数值字段使用业务阈值、用户确认分位点、极端区间、缺失值和 `toad` chi 分箱；类别字段使用确认业务分组、低频合并和缺失组。所有候选必须计算覆盖、坏账率、Lift、坏客户捕获和好客户误伤。

多特征规则只来自受限 CART 的根到叶 AND 路径。路径最多 3 个特征，并须回到统一分析单位重新计算全部指标；不能把树模型的训练分数直接当规则效果。

## 5. 组合与策略逻辑

阶段 2 不重新发掘规则，只在合格候选中按确认的 Lift 定义降序排列。用户确认 TopN 后按顺序级联：任一规则命中即拒绝。每一层只对上层未拒绝样本计算新增拒绝、边际坏客户捕获、边际好客户误伤和剩余样本，不能相加独立规则指标。

阶段 3 将级联规则视为新准入策略。新策略与旧策略在同一分析总体对齐后划分为 `both_pass`、`both_reject`、`swap_in`、`swap_out`。`swap_out` 在旧策略通过且有成熟表现时可直接观察；`swap_in` 通常无直接表现，必须标为“估计”或“不可观测”，不得伪装成直接观测。

## 6. 报告边界

报告记录确认口径、数据质量、候选与组合依据、新旧策略取舍、Swap 可观测性和局限。报告可以给出证据支持的结论与风险披露，但不得自动产生上线、放量或审批决定。
