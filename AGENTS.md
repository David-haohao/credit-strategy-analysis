# 信贷规则策略执行约束

## 固定顺序

1. 确认数据、产品、分析场景、样本、颗粒度、标签和字段映射。
2. 挖掘单规则：IV、极端区间、分位点、缺失值、业务阈值和受限 CART 路径。
3. 通过确认的最小样本和命中限制后，按 Lift 降序选取确认的 TopN，级联拒绝。
4. 评估新策略整体效果；有旧策略时做 Swap。
5. 仅基于阶段产物和 manifest 生成最终报告。

## 不可推断项

不得自行决定产品、样本口径、分析颗粒度、`id_cols`、字段映射、标签、成熟规则、金额口径、分位点、分箱、最小样本、最小命中率、TopN、业务阈值或 `swap_in` 估计方法。

## 边界

- 规则组合固定为 Lift 排序后的级联拒绝，任一命中即拒绝。
- CART 只保留单条路径不超过 3 个特征的候选规则。
- CSV 使用 UTF-8-SIG；JSON、YAML、Markdown、HTML 使用 UTF-8；每次运行写入 manifest。
- 不生成生产上线、灰度或审批结论。
