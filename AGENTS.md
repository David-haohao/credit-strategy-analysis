# 信贷规则策略分析协作规范

## 目标与边界

本仓库的正式工作流只有五段：数据与产品确认、单规则挖掘、Lift TopN 规则组合、策略效果与 Swap、最终报告。仅支持离线分析，不生成线上配置、上线审批或自动发布建议。

不要恢复监控、分数截断、分路由、通用漏斗等并行模块；它们不是本技能入口。

## 必须确认的事实

执行任何阶段前，用户必须确认并写入独立确认凭证：

- 产品形态、还款方式与分析场景；
- 样本口径、时间窗口与分析颗粒度；
- `id_cols`、标签/特征/输出颗粒度，以及跨颗粒度聚合规则；
- 字段语义映射、正样本定义、成熟规则与候选特征；
- 金额指标分子、分母、公式和过滤条件；
- 规则候选参数；规则组合的 Lift 定义、TopN 与级联拒绝语义；
- 旧策略存在时的旧/新决策字段，以及 `swap_in` 的估计或不可观测处理。

不得猜测、替用户选择或将候选映射直接用于分析。循环贷的标签可能在借据或支用层，贷前特征可能在人层；未确认聚合规则时停止。

## 文档读取

按 `SKILL.md` 的五段路由按需读取。阶段 0 是其余阶段的必要前提；阶段 4 只读取前四阶段的已完成产物和 manifest，不重新计算、不补猜口径。

## 工程约束

- 所有文本、配置和输出使用 UTF-8。
- `toad` 是 IV、WOE 与分箱任务的必需依赖；缺失时阶段 1 阻断。
- CART 只提取根到叶的 AND 路径，任何路径最多 3 个特征。
- 规则组合仅允许确认后的 Lift 降序 TopN 和“任一命中即拒绝”级联语义。
- 金额、成熟风险和 Swap 的计算必须标明分母、样本范围与可观测性。
- 当前算法入口只是校验骨架，不得将其输出描述为已完成挖掘或策略评估。

## 验证与提交

每次修改后运行 Python 测试、YAML/JSON 解析和五个 CLI 的 `--help`。检查 Git diff 不包含 `design-framework-v2.md`；它是用户保留的未跟踪草稿。验证完成后提交并推送 `main`。
